事前準備 — アカウント & ツール
必要なものを揃える
  • GitHub アカウント（無料）— github.com
  • Vercel アカウント（無料）— vercel.com
  • Git インストール済み — git-scm.com
  • Discord Developer Portal の Client ID / Secret 取得済み
  • Discord Bot Token 取得済み
💡
Vercel は GitHub アカウントで直接サインアップできます。新規登録は不要です。
▲ vercel.com にアクセス → 「Continue with GitHub」をクリック
2
GitHub にリポジトリをアップロード
プロジェクトをGitHubへPush

Vercel はGitHubリポジトリから直接デプロイします。まずローカルのプロジェクトをGitHubへ上げます。

① GitHubで新しいリポジトリを作成

  1. github.com/new にアクセス
  2. Repository name: discord-auth-bot-py
  3. Private または Public を選択
  4. 「Create repository」をクリック

② ローカルからPush（PowerShell / ターミナル）

cd C:\Users\PZ\Downloads\discord-auth-bot-py\discord-auth-bot

git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/discord-auth-bot-py.git
git push -u origin main
▲ Push完了後のGitHubリポジトリ — web/, bot/, vercel.json が確認できればOK
⚠️
.env ファイルは絶対にPushしない！ 環境変数は Vercel のダッシュボードで設定します。
3
Vercel にプロジェクトをImport
GitHubリポジトリを接続する
  1. vercel.com/new にアクセス
  2. 「Import Git Repository」セクションで discord-auth-bot-py を探す
  3. 右側の 「Import」 ボタンをクリック
▲ リポジトリ一覧から discord-auth-bot-py を選択して Import
ℹ️
リポジトリが表示されない場合は「Adjust GitHub App Permissions」をクリックしてアクセス許可を追加してください。
4
プロジェクト設定 (Configure Project)
フレームワーク・ルートディレクトリを指定
項目設定値説明
Framework PresetOtherフレームワークなし（Python Serverless）
Root Directorywebwebフォルダをルートに指定
Build Command（空欄）ビルド不要
Output Directory（空欄）Vercelが自動検出
Install Commandpip install -r requirements.txt依存パッケージインストール
▲ Configure Project — Root Directory を "web" に変更するのが最重要
⚠️
Root Directory は必ず web に設定！ デフォルトのままだと api/ が見つからずエラーになります。

vercel.json の内容を確認（すでに存在するはずです）

{
  "version": 2,
  "builds": [
    { "src": "api/*.py", "use": "@vercel/python" },
    { "src": "static/**", "use": "@vercel/static" }
  ],
  "routes": [
    { "src": "/api/(.*)", "dest": "/api/$1.py" },
    { "src": "/(.*)", "dest": "/static/$1" }
  ]
}
5
環境変数 (Environment Variables) の設定
シークレット情報をVercelに登録

Deploy前 または Settings → Environment Variables から以下を登録します。

キー名値の例
DISCORD_CLIENT_ID 123456789012345678 必須
DISCORD_CLIENT_SECRET aBcDeFgH1234567890aBcDeF 必須
DISCORD_BOT_TOKEN MTIz...（長いトークン） 必須
WEB_URL https://あなたのプロジェクト.vercel.app 必須
VERIFIED_ROLE_ID 987654321098765432 任意
SUPABASE_URL https://xxxx.supabase.co 必須
SUPABASE_KEY eyJhbGc...（Supabaseキー） 必須
▲ Settings → Environment Variables → キーと値を入力して「Save」
💡
環境変数を追加・変更したあとは必ず 「Redeploy」 しないと反映されません。

どこで値を取得するか

  • DISCORD_CLIENT_ID / SECRET → Discord Developer Portal → OAuth2
  • DISCORD_BOT_TOKEN → Developer Portal → Bot → Reset Token
  • WEB_URL → Vercel デプロイ完了後に表示されるURL
  • SUPABASE_URL / KEY → Supabase → Project Settings → API
6
Discord Developer Portal の設定
Redirect URI を登録する
  1. discord.com/developers/applications を開く
  2. 対象アプリを選択 → 左メニュー 「OAuth2」
  3. 「Redirects」セクションの 「Add Another」 をクリック
  4. 以下のURLを入力して「Save Changes」
https://あなたのプロジェクト名.vercel.app/api/callback
⚠️
このURLは WEB_URL 環境変数と完全に一致させてください。末尾スラッシュの有無も要注意。

OAuth2 Scopes の確認（認証リンク生成時に必要）

  • identify — ユーザーID・名前取得
  • email — メールアドレス取得（必須）
  • guilds.join — サーバーに自動参加
7
デプロイ実行 & 確認
いよいよ本番デプロイ！
  1. ステップ4・5の設定後 → 「Deploy」 ボタンをクリック
  2. ビルドログが流れ始める（1〜3分）
  3. 🎉 「Congratulations!」が表示されればデプロイ完了
  4. 表示されたURLにアクセスして動作確認
▲ 緑のチェックマーク → デプロイ成功！URLをコピーして WEB_URL に設定する

動作確認チェックリスト

  • https://xxx.vercel.app/verify?guild=ID&user=USER_ID でページが表示される
  • 「認証を開始する」→ Discord OAuth2 画面にリダイレクトされる
  • 認証後 /api/callback が呼ばれ成功ページが表示される
  • ログチャンネルに認証ログが届く

エラーが出たときの対処

エラー原因・対処
invalid_requestURLパラメータ不足。認証リンクを再生成
token_exchange_failedCLIENT_ID/SECRET が間違い。環境変数を確認
state_mismatchCookieが消えた。ブラウザの設定を確認
email_domain_blockedブロック済みドメイン（usagica.com等）のメール
500 Internal Server ErrorVercel Logs → Functionsタブでエラー詳細を確認
8
コードの更新（再デプロイ）
コード変更を反映させる方法

GitHubの main ブランチにpushするだけで自動的に再デプロイされます。

git add .
git commit -m "fix: メールドメインブロック追加"
git push
💡
Vercel ダッシュボード → Deployments タブでリアルタイムのビルドログを確認できます。

🎉 デプロイ完了！

すべてのステップが完了すれば Discord Auth Bot が稼働中です。
問題が発生した場合は Vercel の Functions ログ を確認してください。
